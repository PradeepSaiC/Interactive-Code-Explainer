# Interactive Code Explainer

**Interactive Code Explainer** is a web app that takes code, splits it into logical blocks, and provides clear, beginner-friendly explanations for each block. It is designed to help users understand code step-by-step, regardless of their programming background.

## Features

- **AI-Powered Block Splitting:** Uses Google Gemini (LLM) to split code into logical, contiguous blocks that are best suited for explanation.
- **Block-by-Block Explanations:** Each code block receives a detailed, beginner-friendly explanation generated by Gemini.
- **Any Language Support:** Works with Python, JavaScript, Java, C, C++, and more.
- **Robust Mapping:** Ensures every line of code is mapped to a block, preserving all whitespace and empty lines.
- **Modern, Responsive UI:** Clean, accessible interface with light/dark mode, mobile support, and animated block highlighting.
- **Production Ready:** Built with Next.js and React, ready for deployment (e.g., Vercel).

---

## How It Works

### 1. Code Input

- Users paste or write code in the editor.
- The app supports multiple languages (Python, JavaScript, Java, C, C++).

### 2. Block Splitting (Gemini LLM)

- When the user clicks "Generate Explanation," the app sends the code to Gemini with a carefully engineered prompt.
- The prompt instructs Gemini to split the code into logical, explainable blocks, preserving all formatting and whitespace.
- If Gemini returns only one block, a fallback prompt requests finer-grained splitting.

### 3. Block Mapping

- The app robustly maps Gemini's blocks back to the original code, handling:
  - Exact matches (including all whitespace and empty lines)
  - Fuzzy matches (ignoring whitespace or empty lines)
  - Blocks that are only empty lines
  - Consecutive blank lines
  - Any programming language
  - Never skipping or duplicating lines, always preserving order

### 4. Explanation Generation

- Each block is sent to Gemini with a prompt to generate a clear, beginner-friendly explanation.
- Explanations are returned as a JSON array and displayed alongside the code.
- If explanation fails, the user sees "Could not generate an explanation for this block."

### 5. User Experience

- Users can navigate between blocks, with the current block highlighted in the code editor.
- The UI is responsive, accessible, and includes a loader overlay during AI operations.
- A "Beta" badge is shown to indicate the app is in testing.

## Codebase Structure

```
src/
  app/
    page.tsx         # Main app logic, Gemini API calls, block mapping, UI state
    globals.css      # Custom styles for highlighting, layout, and responsiveness
    layout.tsx       # App layout and theming
  components/
    CodePanel.tsx        # Code editor and block highlighting logic
    ExplanationPanel.tsx # Renders AI-generated explanations in Markdown
    codeBlocks.ts        # (Legacy) Tree-sitter-based block extraction (not used in production)
```

### Key Files

- **src/app/page.tsx:**  
  - Main React component.
  - Handles user input, Gemini API calls, block mapping, and UI state.
  - Implements robust logic for mapping Gemini's output to the original code.

- **src/components/CodePanel.tsx:**  
  - Renders the code editor (CodeMirror).
  - Handles block highlighting, navigation, and theming.

- **src/components/ExplanationPanel.tsx:**  
  - Displays AI-generated explanations using ReactMarkdown with syntax highlighting.

- **src/components/codeBlocks.ts:**  
  - (Legacy) Tree-sitter-based code block extraction for various languages.
  - Not used in production; all block splitting is now handled by Gemini.

- **src/app/globals.css:**  
  - Custom styles for block highlighting, loader overlay, and responsive design.

## Customization & Environment

- **Gemini API Key:**  
  Set your Gemini API key in the environment variable `NEXT_PUBLIC_GEMINI_API_KEY`.

- **Styling:**  
  Uses Tailwind CSS and custom styles for a modern, accessible look.

- **Accessibility:**  
  All interactive elements are keyboard accessible and screen reader friendly.

## Getting Started

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Set your Gemini API key:**
   - Create a `.env.local` file:
     ```
     NEXT_PUBLIC_GEMINI_API_KEY=your-gemini-api-key-here
     ```

3. **Run the development server:**
   ```bash
   npm run dev
   ```

4. **Open the app:**
   - Visit [http://localhost:3000](http://localhost:3000) in your browser.

---

## Deployment

- Deploy easily to [Vercel](https://vercel.com/) or any platform that supports Next.js.
- Make sure to set the `NEXT_PUBLIC_GEMINI_API_KEY` environment variable in your deployment settings.

## Notes

- **No code or explanation is ever stored server-side.** All processing is done client-side except for Gemini API calls.
- **ESLint is disabled** in main files to prevent linter errors from blocking production builds.
- The app is in **Beta**—feedback and contributions are welcome!
